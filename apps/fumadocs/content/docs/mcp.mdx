## What is the Cobalt MCP server?

A hosted [Model Context Protocol (MCP)](https://modelcontextprotocol.io) server that gives AI agents read-only access to your Cobalt financial data — accounts, transactions, investments, brokerage positions, and more.

- Authenticated via **OAuth** for secure, scoped access to your account.
- All queries are scoped to the authenticated user via **Postgres row-level security**.
- Accessible from any MCP client that supports HTTP-hosted servers.

**Server URL:**

```
https://api.cobaltpf.com/api/mcp
```

## Authentication

The MCP server uses **OAuth** via Better Auth. When you connect for the first time, your client will redirect you to sign in with your Cobalt account.

- Each client (Cursor, Claude Code, VS Code, etc.) registers as an OAuth application with Cobalt.
- After sign-in, the OAuth token is cached by your client.
- All queries are automatically scoped to your user via row-level security — you can only access your own financial data.

## Security

- The agent database role has **SELECT-only** permissions. Write operations are rejected at both the application and database level.
- Row-level security (RLS) policies scope every query to the authenticated user.
- Queries have a **10-second timeout** to prevent runaway operations.
- Dangerous SQL patterns (write keywords, `SET ROLE`, unsafe functions) are rejected before execution.

<Callout type="info">
  The Cobalt MCP server is read-only. There are no write tools — your data
  cannot be modified through MCP.
</Callout>

Manual installation:

<Steps>
  <Step>
    Open the command palette and type "Cursor Settings"
  </Step>
  <Step>
    Under "Tools & MCP" click "New MCP Server"
  </Step>
  <Step>
    Paste the following JSON into the configuration file:

    ```json
    {
      "mcpServers": {
        "cobalt": {
          "url": "https://api.cobaltpf.com/api/mcp"
        }
      }
    }
    ```

  </Step>
  <Step>
    Save the file. Cursor will prompt you to authenticate — follow the browser flow to sign in with your Cobalt account.
  </Step>
  <Step>
    You may need to restart Cursor to load the new configuration.
  </Step>
</Steps>

  </Step>
  <Step>
    Type "MCP: Add Server" and select HTTP
  </Step>
  <Step>
    Enter the URL `https://api.cobaltpf.com/api/mcp` and name it `Cobalt`
  </Step>
  <Step>
    Start the server via "MCP: List Servers" > select Cobalt > "Start Server"
  </Step>
  <Step>
    When prompted to authenticate, click "Allow" and complete the sign-in flow in your browser.
  </Step>
</Steps>

## Available tools

The Cobalt MCP server exposes 5 tools for schema discovery and data querying:

### 1. `cobalt_list_tables`

Returns all available database tables and their descriptions. **Call this first** to discover what data is available.

**Input:** None

**Example response:**

```json
{
  "tables": [
    {
      "name": "transaction",
      "description": "Bank transactions (posted and pending)"
    },
    {
      "name": "bank_account",
      "description": "Individual bank accounts within connections"
    },
    {
      "name": "investment_position",
      "description": "Current investment holdings per account"
    }
  ]
}
```

### 2. `cobalt_describe_table`

Get column names, data types, and nullability for a specific table. Use after `cobalt_list_tables` to understand table structure before writing queries.

| Parameter | Type   | Description                          |
| --------- | ------ | ------------------------------------ |
| `table`   | string | Table name from `cobalt_list_tables` |

### 3. `cobalt_get_relationships`

Get foreign key relationships between tables. Useful for understanding how to JOIN tables in queries.

**Input:** None

### 4. `cobalt_query`

Execute a read-only SQL query against your financial data. Results are automatically scoped to the authenticated user via row-level security.

| Parameter | Type   | Description                               |
| --------- | ------ | ----------------------------------------- |
| `query`   | string | SQL query (SELECT or WITH only)           |
| `limit`   | number | Max rows to return (default 100, max 500) |

**Example:**

```sql
SELECT name, amount, date
FROM transaction
WHERE date >= '2026-01-01'
ORDER BY date DESC
LIMIT 10
```

**Query safeguards:**

- Only `SELECT` and `WITH` (CTE) queries are allowed
- Write operations (`INSERT`, `UPDATE`, `DELETE`, `DROP`, etc.) are rejected
- Multi-statement queries are rejected
- Dangerous functions (`pg_read_file`, `dblink`, `pg_sleep`, etc.) are blocked
- `SET ROLE` escalation is blocked
- A `LIMIT` is automatically appended if missing (default 500)
- Queries have a 10-second timeout

### 5. `cobalt_get_session_subject`

Returns the Cobalt user ID for the current OAuth session. Useful for debugging authentication.

**Input:** None

## Troubleshooting

If your MCP client cannot connect or tools fail to run:

**Restart your client** — After installation, you may need to restart your client or CLI tool to load the new MCP configuration.

**Check the server URL** — Make sure the URL is set to `https://api.cobaltpf.com/api/mcp`.

**Re-authenticate** — If your token has expired, re-authenticate by restarting the MCP connection in your client.

**Query rejected** — Only `SELECT`/`WITH` queries are allowed. Check for accidental write keywords in your query.

**Empty results** — Row-level security scopes data to your user. Verify you have financial accounts linked to your Cobalt account.

**Timeout** — Queries have a 10-second limit. Add `WHERE` filters or reduce the result set with `LIMIT`.
